Installation Guide

Get ForgePulse up and running in your Laravel 12 application in minutes.

Requirements

  • PHP 8.3, 8.4, or 8.5
  • Laravel 12
  • Livewire 4

Step 1: Install via Composer

Install the package using Composer: 

composer require alizharb/forgepulse

Step 2: Publish Configuration

Publish the configuration file to customize ForgePulse: 

php artisan vendor:publish --tag=forgepulse-config

This creates config/forgepulse.php where you can configure:

  • Execution settings (timeout, retries, queue)
  • Role-based permissions
  • Template storage
  • Event hooks
  • UI preferences

Step 3: Run Migrations

Publish and run the database migrations: 

php artisan vendor:publish --tag=forgepulse-migrations
php artisan migrate

This creates 4 tables:

  • workflows - Workflow definitions
  • workflow_steps - Individual workflow steps
  • workflow_executions - Execution tracking
  • workflow_execution_logs - Detailed step logs

Step 4: Publish Assets (Optional)

Optionally publish views, assets, and language files: 

# Publish Blade views
php artisan vendor:publish --tag=forgepulse-views

# Publish CSS/JS assets
php artisan vendor:publish --tag=forgepulse-assets

# Publish language files
php artisan vendor:publish --tag=forgepulse-lang

Step 5: Configure Policies (Optional)

If you want to customize authorization, register the policy in your AuthServiceProvider: 

use AlizHarb\ForgePulse\Models\Workflow;
use AlizHarb\ForgePulse\Policies\WorkflowPolicy;

protected $policies = [
    Workflow::class => WorkflowPolicy::class,
];

Step 6: Team Integration (Optional)

ForgePulse supports optional team integration. To enable it, update your configuration: 

'teams' => [
    'enabled' => true,
    'model' => \App\Models\Team::class,
],

Ensure your teams table exists before running migrations.

Step 7: Set Up Queue (Recommended)

For async workflow execution, configure your queue driver in .env: 

QUEUE_CONNECTION=redis

Then run the queue worker: 

php artisan queue:work
